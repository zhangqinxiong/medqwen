## 🚀 模块三：Unsloth 16-bit 混合精度训练脚本

由于 Qwen2-1.5B 极其轻量，加载仅需 ~3GB 显存，我们在 3060 上**不需要开启 4-bit 量化**，直接采用 **16-bit（bfloat16 或 float16）进行 LoRA 微调**。这样可以完美保留模型的逻辑和语言通顺度。

### 给其他 AI 的提示词：

> "请基于 **Unsloth** 框架，编写一个用于微调 **Qwen2-1.5B-Instruct** 的 Python 训练脚本 `train.py`。读取上一步生成的 `train_med.json` 和 `eval_med.json`。请严格按照以下参数进行配置：
> **1. 模型与LoRA配置：**
> * 使用 `FastLanguageModel.from_pretrained` 加载 `"Qwen/Qwen2-1.5B-Instruct"`。
> * `max_seq_length = 1024`，`load_in_4bit = False`（直接使用 16-bit 提升医疗文本生成质量）。
> * 调用 `get_peft_model` 开启 LoRA。配置：`r = 16`, `lora_alpha = 32`, `lora_dropout = 0`。
> * `target_modules` 覆盖：`["q_proj", "k_proj", "v_proj", "o_proj", "gate_proj", "up_proj", "down_proj"]`。
> 
> 
> **2. 训练器（SFTTrainer）与超参数：**

> * `gradient_accumulation_steps = 4`（等效总 Batch Size = 16）。
> * `learning_rate = 2e-4`, `lr_scheduler_type = "cosine"`, `weight_decay = 0.01`。
> * `num_train_epochs = 3`。
> * 根据 3060 显卡代际，设置 `fp16 = True` 或 `bf16 = True`（推荐优先使用混合精度）。
